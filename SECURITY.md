# Security Policy

## Supported Versions

Currently, only the latest version of **TAJ AI** is actively maintained and supported with security updates.

| Version        | Supported |
| -------------- | --------- |
| Latest         | ✅ Yes     |
| Older versions | ❌ No      |

---

## Reporting a Vulnerability

If you discover a security vulnerability in **TAJ AI**, please report it responsibly.

Do **NOT** create a public GitHub issue for security vulnerabilities.

Instead, report it privately by contacting the project maintainer.

📧 Contact Email: [24tec2csml175@vgu.ac.in](24tec2csml175@vgu.ac.in)

Please include the following information in your report:

* Description of the vulnerability
* Steps to reproduce the issue
* Potential impact
* Screenshots or proof of concept (if possible)

---

## Responsible Disclosure

We request that security researchers follow responsible disclosure practices:

* Allow time for the vulnerability to be fixed before public disclosure.
* Do not exploit the vulnerability beyond what is necessary for testing.
* Do not access or modify data without permission.

---

## Security Best Practices

To ensure security while using or reviewing this project:

* Do not expose API keys or secrets.
* Keep dependencies updated.
* Use environment variables for sensitive configuration.
* Avoid committing private credentials.

---

## Scope

This security policy applies to the **TAJ AI repository and its codebase**.

Unauthorized use, cloning, or redistribution of this project is prohibited according to the project license.

---

## Acknowledgements

We appreciate responsible security researchers who help improve the safety and reliability of **TAJ AI**.
